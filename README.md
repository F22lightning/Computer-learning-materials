# Computer Parts Learning Application

เว็บแอปพลิเคชันสำหรับใช้เป็นสื่อการเรียนการสอนเกี่ยวกับส่วนประกอบของคอมพิวเตอร์ (Computer Components) ออกแบบมาเพื่อให้ผู้ใช้งานสามารถโต้ตอบ (Interactive) ด้วยการคลิกเลือกหมายเลขอุปกรณ์หน้าชั้นเรียน หรือชิ้นส่วนที่สนใจ เพื่อดูรายละเอียดการทำงานและวิดีโออธิบายของอุปกรณ์ชิ้นนั้นๆ

## คุณสมบัติ (Features)
- **Interactive UI:** ปุ่มกดหมายเลขขนาดใหญ่ ชัดเจน เหมาะสำหรับการนำไปติดป้ายหมายเลขไว้ที่อุปกรณ์จริง
- **Fullscreen Detail View:** เมื่อคลิกที่ปุ่ม จะแสดงหน้าต่างรายละเอียดแบบเต็มจอ พร้อมข้อมูลชื่อภาษาไทย ชื่อภาษาอังกฤษ และคำอธิบายการทำงาน
- **Video Support:** รองรับการเล่นวิดีโออธิบายแบบฝังตัว (Embedded Video) พร้อมปุ่มควบคุม และ Responsive Design สัดส่วนสมจริง
- **Dockerized:** พร้อมใช้งานทันทีผ่าน Docker ทำให้การติดตั้งและนำไปเปิดใช้งานบนเครื่องต่างๆ เป็นเรื่องง่าย

## รายละเอียดอุปกรณ์ที่มีในระบบ
1. ฮาร์ดดิสก์ (Hard Disk Drive)
2. แรม (Random Access Memory)
3. เมนบอร์ดและซีพียู (Motherboard & CPU)
4. พัดลมระบายความร้อน (Cooling Fan)
5. ออปติคอลไดรฟ์ (Optical Drive - CD/DVD ROM)
6. ลำโพงของคอมพิวเตอร์ (Computer Speaker)
7. โซลิดสเตตไดรฟ์ (Solid State Drive - SSD)

## การควบคุมผ่านคีย์บอร์ด (Keyboard Shortcuts)
เพื่อความสะดวกในการใช้งานหรือการสาธิตหน้าชั้นเรียน คุณสามารถใช้ปุ่มบนคีย์บอร์ด (โดยเฉพาะ Numpad) ได้ดังนี้:
- **ปุ่ม 1-7:** เลือกดูอุปกรณ์ (แสดงผลแบบ Overlay พร้อมคำอธิบาย)
- **ปุ่ม 8:** ขยายวิดีโอแบบเต็มจอ (Manual Fullscreen)
- **ปุ่ม 9:** ออกจากโหมดเต็มจอ หรือ กลับหน้าหลัก (Exit/Back)
- **ปุ่ม + (บวก):** เลื่อนไปดูอุปกรณ์ "ชิ้นถัดไป"
- **ปุ่ม - (ลบ):** ย้อนกลับไปดูอุปกรณ์ "ชิ้นก่อนหน้า"
- **ปุ่ม 0:** เล่น (Play) หรือ หยุด (Pause) วิดีโอ
- **ปุ่ม * (ดอกจัน):** กลับสู่หน้าหลัก (Home)

## รูปแบบการนำไปใช้งาน (Real-world Application)
สื่อการสอนนี้ถูกออกแบบมาเพื่อใช้งานร่วมกับ **"บอร์ดชิ้นงานจริง"** โดยมีลักษณะการติดตั้งดังนี้:
1. **บอร์ดอุปกรณ์:** นำอุปกรณ์คอมพิวเตอร์จริง (HDD, RAM, Mainboard, etc.) มาติดตั้งบนแผงโชว์
2. **หน้าจอแสดงผล:** ติดตั้งมอนิเตอร์ไว้ข้างชุดอุปกรณ์จริง เพื่อแสดงวิดีโออธิบาย
3. **ชุดควบคุม:** ต่อพ่วง **Numpad แยก** สำหรับให้ผู้เรียนกดหมายเลขที่ตรงกับป้ายกำกับบนอุปกรณ์ชิ้นนั้นๆ
4. **ประสบการณ์การเรียนรู้:** เมื่อกดเลข วิดีโอจะแสดงผลแบบ **Fullscreen** ทันที เพื่อให้การเรียนรู้น่าสนใจและเห็นภาพชัดเจนที่สุด

## การติดตั้งลงในเครื่องที่จะใช้งานจริง (Deployment Guide)
คุณสามารถนำโปรเจคนี้ไปรันบนเครื่องคอมพิวเตอร์ที่ต่อกับหน้าจอแสดงผลได้ 2 วิธี:

### วิธีที่ 1: ใช้ Docker (แนะนำที่สุด)
วิธีนี้จะช่วยให้ระบบทำงานได้เสถียรที่สุด เพราะมีการตั้งค่าเว็บเซิร์ฟเวอร์ (Nginx) ไว้ให้เรียบร้อยแล้ว
1. ติดตั้ง [Docker Desktop](https://www.docker.com/products/docker-desktop/) ลงในเครื่องที่จะใช้งาน
2. เปิด Terminal (หรือ PowerShell) แล้ว Clone โปรเจคลงมา:
   ```bash
   git clone https://github.com/F22lightning/Computer-learning-materials.git
   cd Computer-learning-materials
   ```
3. สั่งรันด้วยคำสั่ง:
   ```bash
   docker-compose up -d --build
   ```
4. เปิดเบราว์เซอร์ไปที่: `http://localhost:9090`

### วิธีที่ 2: ใช้ Web Server แบบง่าย (เช่น Python หรือ VS Code)
หากไม่สะดวกใช้ Docker สามารถรันผ่านเซิร์ฟเวอร์จำลองทั่วไปได้:
1. Clone โปรเจคลงมาที่เครื่อง
2. รันคำสั่งภาษา Python (หากมีติดตั้งไว้):
   ```bash
   python -m http.server 8000
   ```
3. หรือใช้ Extension **"Live Server"** บน VS Code เพื่อรันหน้าจอขึ้นมา

**หมายเหตุ:** การดับเบิลคลิกไฟล์ `index.html` เพื่อเปิดตรงๆ อาจทำให้เบราว์เซอร์บางตัวบล็อกวิดีโอหรือฟังก์ชันจดจำปุ่มกดได้ แนะนำให้รันผ่านเซิร์ฟเวอร์ตาม 2 วิธีข้างต้นครับ

## วิธีการติดตั้งและใช้งาน (Original Docs)

### วิธีที่ 1: การใช้งานแบบ Local (ไม่ต้องติดตั้งโปรแกรมเสริม)
1. ดาวน์โหลดหรือ Clone Repository นี้ลงมาที่เครื่องคอมพิวเตอร์ของคุณ
```bash
git clone https://github.com/F22lightning/Computer-learning-materials.git
```
2. ดับเบิลคลิกที่ไฟล์ `index.html` เพื่อเปิดใช้งานผ่าน Web Browser (เช่น Chrome, Edge, Firefox) ได้ทันที

### วิธีที่ 2: การใช้งานผ่าน Docker (แนะนำสำหรับการจำลอง Server)
**ข้อกำหนดเบื้องต้น:** เครื่องของคุณต้องติดตั้ง Docker และ Docker Compose เรียบร้อยแล้ว

1. เปิด Command Prompt หรือ Terminal แล้วเข้าไปที่โฟลเดอร์ของโปรเจกต์
2. รันคำสั่งต่อไปนี้เพื่อสร้างและเปิดคอนเทนเนอร์:
```bash
docker-compose up -d --build
```
3. เปิด Web Browser แล้วเข้าไปที่ `http://localhost:9090`

## โครงสร้างไฟล์
- `index.html` - โครงสร้างหน้าเว็บหลัก
- `style.css` - ไฟล์สำหรับตกแต่งความสวยงามและการจัดวาง Layout
- `script.js` - ไฟล์ JavaScript สำหรับควบคุมการทำงาน, การเปลี่ยนหน้า, โหลดข้อมูล และจัดการวิดีโอ
- `Dockerfile` & `docker-compose.yml` - ไฟล์สำหรับการรัน Web Server ผ่าน Docker (ใช้ Nginx)
- `*.mp4` - ไฟล์วิดีโอประกอบสื่อการสอน
